# M1.6 — Uncertainty as a runtime monitor (selective prediction) (spec)

## Question
M1.2–1.5 showed LeWM's uncertainty can't *improve* a decision. M1.6 asks the complement: can it tell you
**when to abstain** from the model's prediction? If yes, the thread gains a constructive positive — and a
clean dividing line: *uncertainty is for knowing you don't know (detection), not for deciding better.*

## Method (per-transition, no retrain)
For each transition compute the true prediction error `e = ‖predict − true_next_latent‖` and two
uncertainty signals:
- **MC-dropout variance** of the predictor (predictive-error facet, M1.1: Pearson +0.41 in-dist).
- **Shell deviation** `|‖emb‖ − shell|` (OOD/epistemic facet, M1.1: flat vs in-dist error → orthogonal).

Rank items by a signal (ascending), keep the most-confident coverage `c`, measure mean `e` on the kept set
→ **risk–coverage curve**; summarize by **AURC** (lower = better).

## Two failure modes, reported separately (so the win isn't a trivial OOD-sort)
- **In-distribution:** clean Push-T transitions; `e` varies at contacts. Can **MC-variance** abstain from
  the hard ones? (No OOD tell — the non-trivial monitor claim.)
- **Shift:** the current frame is corrupted with Gaussian noise (`NOISE_SIGMA`); the model predicts from
  the corrupted latent, error vs the clean next. Can the **shell** signal abstain from these?

## Signals compared
`MC-variance`, `shell`, `combined` (z-score sum within the pool), vs **oracle** (rank by true `e`) and
**random** (flat at mean `e`).

## Eval (pre-registered)
AURC table over two pools — **in-dist** (clean only) and **mixed** (clean + corrupted):

- **POSITIVE** if uncertainty-gated AURC ≪ random, AND the facets are **complementary**: `MC-variance`
  wins in-dist (shell ≈ random there), `shell` wins on the mixed/shift pool, and `combined` ≥ both,
  approaching oracle. → a free, calibrated runtime monitor; the earlier "complementary facets" *analysis*
  becomes a constructive *monitor*.
- **NULL** if no signal beats random (would contradict M1.1 — not expected).

Sanity check printed: mean shell deviation on corrupted vs clean (confirms the corruption is OOD).
