# M1.4 — Learned one-step-surprise head (spec)

## Question
M1.3 found that MC-dropout predictive variance is **flat** across a rollout (statistically tied with
random for scheduling, ~0.05–0.1 regardless of step), yet the **oracle** — looking at the
intrinsically-most-surprising steps — beats fixed-interval by up to ~30% at mid-budget. So the headroom
is real, but MC-dropout can't see where it is.

**M1.4 asks: is one-step surprise predictable from `(state, action)` *before* the next frame is seen,
sharply enough to recover the oracle's scheduling?**
- **Yes** → the thread's first constructive *positive*: a sharp, causal, no-retrain uncertainty that
  makes a JEPA world model's uncertainty actionable for sensing.
- **No** → surprise is intrinsically unpredictable one step ahead here; even a learned signal can't beat
  uniform spacing. A deeper, still-clean negative.

## The head
- **Input:** `(z, a)` — LeWM latent `z` (192-d) ⊕ action-encoded `a` (192-d, via `model.action_encoder`)
  → 384-d.
- **Output:** scalar predicted one-step surprise `ŝ ≈ ‖predict(z, a) − z_next‖`.
- **Architecture:** small MLP (384 → 256 → 128 → 1, ReLU), regression on `log1p(surprise)` (the label has
  a heavy tail — peaks at contacts); exponentiate `ŝ` for the look-trigger. ~150k params.
- **Cost:** frozen LeWM, head only. Trains in seconds on logged labels. **No LeWM retrain.**

## Training data — true latents (the chosen scope, YAGNI)
- **Labels:** the rig's `intrinsic_surprise[t] = ‖predict(true z, a) − true z_t‖`, on TRUE encoded latents.
- **Pairs:** `(true z_{t-1}, a_{t-1}) → intrinsic_surprise[t]` over all steps of the training rollouts.
- **Split:** rollouts split train/val (20 train / 10 eval). Held-out correlation **and** deployment are
  measured on UNSEEN rollouts — no leakage.
- **Deploy mismatch (measured, not hidden):** at deploy the head sees the *maintained* (drifted) latent
  `ẑ_{t-1}`, not the true one. This train/deploy gap is a known limitation; eval part 2 quantifies its
  cost. Drift-aware training is the **M1.5 fallback** — built only if the gap actually hurts.

## Arms
Extend the existing M1.3 comparison `{variance, fixed, random, oracle}` with:
- **`learned`** — causal threshold on the head's predicted surprise `ŝ` (sweep τ for the budget axis),
  deployed on held-out rollouts.
- **`action-mag`** — threshold on `‖a‖` (near-free causal baseline).
- **`latent-drift`** — threshold on `‖ẑ_t − ẑ_{t-1}‖` (near-free causal baseline).

The trivial baselines guard against "the head is no better than a one-liner."

## Eval — pre-registered
1. **Scientific (the core question):** held-out Spearman(`ŝ`, true `intrinsic_surprise`). MC-dropout's
   effective value was ≈0 (flat). "Predictable" = clearly > 0 **and** above the action-mag / latent-drift
   correlations.
2. **Deployment:** `learned` error-vs-budget curve vs fixed-interval and vs the oracle ceiling, on
   held-out rollouts at matched budget.

**Verdict branches:**
- **WIN** — `learned` beats fixed-interval beyond SEM AND closes a meaningful fraction of the
  fixed→oracle gap. *(Sharp causal signal recovers the headroom MC-dropout missed.)*
- **PARTIAL** — eval 1 positive (head predicts surprise on held-out true latents) but deploy degrades to
  ≈ fixed-interval → the drift mismatch dominates → motivates **M1.5** (drift-aware training).
- **NULL** — eval 1 ≈ 0 (head can't predict surprise) → surprise is intrinsically unpredictable one step
  ahead; sensing can't beat uniform here. Clean negative.

## Code structure
- **Refactor `src/active_sense.py`:** wrap the experiment driver under `if __name__ == "__main__":`,
  leaving the rig helpers importable at module level — `rollout`, `encode_all`, `act_encode`,
  `_predict_one`, `tracking_errors`, `variance_sim`, `intrinsic_surprise`, `set_drop`, `sem`. No behavior
  change; M1.3 still runs identically.
- **New `src/surprise_head.py`:** imports the rig, collects rollouts, builds train/val pairs, trains the
  head, runs eval 1 (held-out corr) + eval 2 (deploys `learned` / `action-mag` / `latent-drift` alongside
  `fixed` / `oracle`), prints the verdict, saves a figure (predicted-vs-true surprise scatter + the
  error-vs-budget plot with the `learned` curve added). Colab GPU, same `/content` convention as the
  other M1.x scripts.
- **Tests:** the schedule unit tests stay green. Add a small local test for the pair-construction /
  label-alignment helper (pure index math, no torch) if it factors out cleanly.

## Reuse / cost
Reuses `load_lewm`, `schedules.py`, and the refactored rig. Head training is trivial; the heavy cost is
encoding rollouts (already amortized) + the τ-sweeps. Runtime ≈ M1.3 (a few minutes on Colab GPU).

## Out of scope (YAGNI)
Drift-aware training (→ M1.5, only if deploy shows drift dominates), ensemble heads, LeWM retraining,
spatial / where-to-look sensing.
